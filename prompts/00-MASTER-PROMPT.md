# MASTER PROMPT — 1-qadam (Claude Design'ga birinchi bo'lib shuni yuboring)

---

Ты — senior product-дизайнер и frontend-разработчик. Создай интерактивный HTML-прототип мобильного приложения **«Buxoro POS»** — полного аналога Poster POS (joinposter.com) для ресторанного бизнеса Узбекистана. Прототип потом станет Flutter-приложением (iOS/Android) с Firebase-бэкендом, поэтому структура данных и экранов должна быть чистой и модульной.

## Формат артефакта
- ОДИН файл HTML со встроенным CSS и JS (без внешних библиотек, кроме разрешённых CDN, если нужно; графики рисуй на чистом SVG/canvas).
- Телефонная рамка 390×844 по центру страницы (скруглённая, с «чёлкой»), фон вокруг #F0EEE6.
- Всё приложение — SPA внутри рамки: экраны переключаются JS-роутером (без перезагрузки), состояние хранится в JS-объекте `APP_STATE`.
- Язык интерфейса — русский. Валюта — сум: формат `2 480 000 СУМ` (пробелы-разделители).
- Никаких lorem ipsum и заглушек «в разработке»: каждая кнопка, инпут и переключатель должны реально работать на mock-данных.

## Дизайн-система «Claude Light» (строго соблюдать)
CSS-переменные:
```css
:root{
  --bg:#FAF9F5; --bg-secondary:#F0EEE6; --surface:#FFFFFF; --surface-hover:#F5F4EE;
  --border:#E3E0D5; --border-strong:#D1CEC2;
  --text:#141413; --text-secondary:#6E6B63; --text-tertiary:#9C9A92;
  --accent:#D97757; --accent-hover:#C96442; --accent-soft:#F6E9E2;
  --success:#788C5D; --success-soft:#EDF1E6;
  --danger:#BF4D43; --danger-soft:#F9E9E7;
  --warning:#C9944A; --warning-soft:#F8EFDF;
  --radius-card:16px; --radius-btn:12px;
}
```
- Шрифты: UI — Inter/system-ui; крупные цифры и заголовки экранов — серифный (Georgia/'Tiempos'-стиль). Это фирменный приём Claude: тёплый кремовый фон + терракотовый акцент + серифные заголовки.
- Карточки: белые, бордер 1px var(--border), радиус 16, тени почти нет (0 1px 3px rgba(20,20,19,.06)).
- Основная кнопка: терракотовая (--accent), белый текст, высота 48, радиус 12. Вторичная: белая с бордером. Опасная: красный текст.
- Инпуты: белые, бордер, при фокусе рамка --accent; label сверху 13px --text-secondary.
- Модальные формы — bottom sheet (выезжает снизу, ручка-полоска сверху, скругление 20).
- Тосты успеха: сверху, фон --success-soft, текст --success («Товар сохранён» и т.п.), исчезают через 2,5 с.
- Пустые состояния: emoji 48px + заголовок + описание + CTA-кнопка.
- Бейджи-статусы: pill; «Открыт» — warning-soft, «Закрыт/Проведена» — success-soft, долг/расход — danger.
- НЕЛЬЗЯ: синие/фиолетовые акценты, чисто-белый фон страницы, тяжёлые тени, тёмная тема.

## Архитектура приложения
Нижний таб-бар (64px, 5 вкладок, активная — терракотовая):
1. **Главная** 🏠 — дашборд владельца
2. **Касса** 🛒 — POS-терминал
3. **Меню** 📋 — товары, тех. карты, ингредиенты
4. **Склад** 📦 — остатки, поставки, списания, инвентаризации
5. **Ещё** ⋯ — список: Статистика, Финансы, Маркетинг, Сотрудники, Настройки, Подписка, Приложения

Сверху каждого экрана — app-bar: слева «‹» (если вложенный экран), заголовок, справа контекстные действия. Над app-bar — тонкий баннер-предупреждение (--warning-soft): «Пробный период до 9 июля. Оплатить» (крестик скрывает).

## Что построить в ЭТОМ шаге (шаг 1 из 5)
1. **Каркас**: телефонная рамка, таб-бар, роутер, дизайн-токены, компоненты (кнопки, карточки, бейджи, bottom sheet, тост, пустое состояние) — как переиспользуемые JS-функции/классы.
2. **Экран «Главная»** (дашборд владельца, аналог Poster Boss):
   - Приветствие «Салом, Жафар 👋» + название заведения + дата.
   - Карточка «Сегодня»: крупная выручка (серифный шрифт 32px) + чипы роста (+12% зелёный), под ней сетка 2×2: Прибыль, Чеки, Посетители, Средний чек.
   - График «Выручка за 7 дней» — SVG-линия с точками, терракотовая, с переключателем День/Неделя/Месяц (меняет данные).
   - Блок «Методы оплаты» — 3 горизонтальных прогресс-бара (Наличные/Карточка/Бонусы) с суммами.
   - «Популярные товары» — топ-5 список с количеством.
   - «Открытые заказы (2)» — карточки заказов со временем и суммой, тап → переход на Кассу.
   - Карточка-предупреждение (--warning-soft): «⚠️ Мука ниже лимита: 2,4 кг из 5 кг» → тап ведёт на Склад.
3. **Экран «Ещё»**: профиль (аватар-инициалы, имя, «Владелец»), список разделов с иконками и шевронами (Статистика, Финансы, Маркетинг, Сотрудники, Настройки, Подписка, Приложения), внизу «Выйти» красным. Пункты пока открывают экран-заглушку «Скоро в следующем шаге» (единственное допустимое место заглушки; исчезнет в шагах 3-5).
4. Вкладки Касса/Меню/Склад — пустые экраны с заголовком (наполним в шагах 2-3).

## Критерии качества
- Все цифры отформатированы с пробелами; всё на русском; ни одного англ. слова в UI (кроме «Buxoro POS»).
- Переключатели/чипы/табы реально меняют данные на экране.
- Код структурирован: `MOCK`, `APP_STATE`, `router()`, `ui.*` компоненты — чтобы следующие шаги легко добавляли экраны.
- Выгляди должно дорого и спокойно: много воздуха, аккуратная типографика, единый ритм отступов.

Когда закончишь — кратко перечисли, что готово, и жди следующий шаг (я пришлю модуль «Касса»).
